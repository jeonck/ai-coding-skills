## 목표

특정 사이트의 디자인을 분석해 정적 HTML/CSS로 재현하는 스킬 정의 파일을 직접 작성하고, 실제로 한 페이지에 대해 실행해 결과를 검증합니다.

## 사전 준비

- 클론 대상: 본인이 소유했거나 학습 목적임이 명확한 사이트 (예: 직접 만든 이전 프로젝트 페이지)
- Node.js 또는 정적 HTML 빌드가 가능한 환경

```bash
mkdir -p clone-lab && cd clone-lab
git init -q
```

## 단계별 실행

### 1. 대상 페이지 정보 수집

브라우저 자동화(또는 `curl` + 직접 확인)로 대상 페이지의 HTML 구조와 주요 스타일 값을 기록합니다.

```bash
curl -s https://example.com/ -o reference.html
```

### 2. 스킬 정의 파일 작성

`SKILL.md` 형태로 아래 절차를 문서화합니다.

```markdown
# clone-site-design

트리거: "이 사이트처럼 만들어줘", "디자인 클론해줘"

절차:
1. 대상 URL의 DOM/스타일/스크린샷 수집
2. 색상·타이포그래피·간격을 변수로 추출
3. 헤더/히어로/카드/푸터 단위로 컴포넌트 분해
4. 목표 스택(정적 HTML 등)으로 코드 생성
5. 로컬에서 빌드 후 원본과 스크린샷 비교
```

### 3. 실제로 한 페이지에 적용

수집한 정보를 바탕으로 `index.html` + `style.css`를 작성하고, 로컬 서버로 띄워 원본과 비교합니다.

```bash
python3 -m http.server 8000
```

## 결과 확인

원본 페이지와 결과물을 나란히 열어 레이아웃·색상·타이포그래피가 의도한 수준으로 일치하는지 확인합니다. 완전히 동일할 필요는 없으며, "구조와 패턴을 재현했는가"가 기준입니다.

## 체크리스트

- [ ] `SKILL.md`에 트리거 조건과 5단계 절차가 명시되어 있다
- [ ] 결과물이 로컬 빌드에서 정상적으로 렌더링된다
- [ ] 원본 사이트의 저작권/이용약관을 확인했다
