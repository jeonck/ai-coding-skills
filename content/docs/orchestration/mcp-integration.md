{{< term "MCP" >}}(Model Context Protocol)는 에이전트가 브라우저, 디자인 툴, 사내 시스템 같은 외부 도구를 표준화된 방식으로 호출할 수 있게 하는 프로토콜입니다. 스킬이 "무엇을 어떤 순서로 할지"를 정의한다면, MCP는 그 절차 중 "어떤 외부 시스템을 어떻게 호출할지"를 담당합니다.

## MCP가 스킬을 강화하는 방식

- [사이트 클론 코딩](../../clone-and-scaffold/clone-site-design/) 스킬은 브라우저 자동화 MCP 서버 없이는 대상 페이지의 실제 렌더링 결과를 확인할 수 없습니다.
- [디자인 → 코드 자동화](../../clone-and-scaffold/design-to-code/) 스킬은 디자인 툴 MCP 서버 없이는 컴포넌트 구조와 토큰을 정확히 읽어올 수 없습니다.
- 사내 이슈 트래커, 문서 저장소와 연동하면 스킬이 최신 컨텍스트를 실시간으로 참조할 수 있습니다.

{{< callout type="info" >}}
MCP 서버 도구는 처음부터 전체 스키마가 로드되어 있지 않고 필요할 때 조회해 불러오는 경우가 많습니다. 스킬을 설계할 때 "이 작업에 어떤 MCP 도구가 필요한지"를 미리 목록화해두면, 실행 시점에 필요한 도구를 한 번에 불러와 왕복 횟수를 줄일 수 있습니다.
{{< /callout >}}

## 연동 시 확인할 것

1. **권한 범위** — 이 MCP 서버가 읽기 전용인지, 쓰기·실행 권한까지 있는지
2. **실패 처리** — 외부 서비스가 응답하지 않을 때 스킬이 무한 대기하지 않고 사용자에게 알리는지
3. **부작용 검토** — 외부 시스템에 메시지를 보내거나 파일을 생성하는 등 되돌리기 어려운 동작이 있다면 실행 전 확인 단계를 두는지

{{< callout type="warning" >}}
쓰기 권한이 있는 MCP 도구(메시지 전송, 파일 업로드, 티켓 생성 등)를 훅처럼 사람 확인 없이 자동 실행하도록 연결하지 마세요. 외부 시스템에 영향을 주는 작업은 항상 사람이 결과를 검토할 수 있는 지점을 남겨둬야 합니다.
{{< /callout >}}
