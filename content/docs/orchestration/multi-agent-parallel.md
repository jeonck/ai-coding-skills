여러 파일이나 여러 관점(보안 리뷰 + 성능 리뷰 + 접근성 리뷰)을 동시에 점검해야 할 때, 이를 순서대로 하나씩 처리하면 시간이 선형으로 늘어납니다. 서로 독립적인 작업이라면 서브에이전트를 병렬로 띄워 동시에 처리하는 것이 훨씬 빠릅니다.

## 병렬화가 적합한 경우

- 각 작업이 **서로의 결과에 의존하지 않을 때** (예: 서로 다른 파일에 대한 독립적인 보안 스캔)
- 각 작업의 중간 산출물(로그, 검색 결과)이 많아 메인 대화 맥락에 남기고 싶지 않을 때
- "어느 쪽이 더 나은가"를 비교하기 위해 같은 문제를 다른 방식으로 동시에 풀어봐야 할 때

## 병렬화가 부적합한 경우

- 뒤 단계가 앞 단계의 결과를 반드시 참고해야 하는 순차적 워크플로우
- 같은 파일을 여러 에이전트가 동시에 수정해 충돌이 날 수 있는 작업

{{< callout type="warning" >}}
병렬로 실행한 에이전트가 같은 파일을 동시에 수정하게 하지 마세요. 결과를 병합하는 과정에서 어느 한쪽의 수정이 조용히 사라질 수 있습니다. 파일/디렉터리 단위로 작업을 명확히 분리하세요.
{{< /callout >}}

## 결과 취합 원칙

여러 에이전트의 결과를 한 번에 받으면, 다음 순서로 정리합니다.

1. 각 에이전트의 결과를 **그대로 신뢰하지 말고** 서로 충돌하는 부분이 있는지 먼저 대조
2. 가장 심각하거나 확실한 발견부터 정렬
3. 중복으로 보고된 이슈는 하나로 합치기
