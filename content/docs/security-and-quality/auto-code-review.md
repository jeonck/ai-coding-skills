사람 리뷰어는 매번 같은 관점(네이밍, 중복, 에러 처리, 테스트 커버리지)을 반복해서 확인합니다. 이 반복을 스킬로 고정하면, PR이 올라올 때마다 "리뷰 스킬 실행" 한 줄로 1차 점검을 마칠 수 있습니다.

## 리뷰 스킬에 담아야 할 관점

1. **정확성** — 엣지 케이스, null/빈 값 처리, 동시성 문제
2. **재사용·단순화** — 불필요한 추상화, 중복 로직
3. **효율성** — 불필요한 반복 조회, N+1 문제
4. **테스트 커버리지** — 변경분에 대응하는 테스트가 있는지

{{< callout type="info" >}}
리뷰 스킬은 발견한 문제를 **가장 심각한 순서로, 재현 가능한 실패 시나리오와 함께** 보고하도록 설계하는 것이 핵심입니다. "이 코드는 별로예요"가 아니라 "입력 X일 때 Y가 발생해 Z가 깨집니다" 형태로 고정하면 리뷰 결과를 바로 실행 가능한 수정으로 옮길 수 있습니다.
{{< /callout >}}

## 사람 리뷰와의 관계

자동 코드 리뷰 스킬은 사람 리뷰어를 대체하지 않습니다. 반복적이고 규칙 기반인 1차 점검(스타일, 명백한 버그 패턴, 누락된 null 체크)을 먼저 걸러내, 사람 리뷰어가 아키텍처 결정이나 비즈니스 로직 같은 고차원 판단에 집중할 수 있게 하는 것이 목적입니다.

{{< callout type="warning" >}}
리뷰 스킬이 "이상 없음"을 보고했다고 해서 병합을 자동 승인하지 마세요. 스킬은 알려진 패턴만 확인할 뿐, 요구사항 충족 여부나 팀 컨벤션과의 정합성까지는 판단하지 못합니다.
{{< /callout >}}
