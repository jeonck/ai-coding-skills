{{< term "스킬" >}}은 "매번 다시 설명할 필요 없이, 이름만 부르면 실행되는 저장된 워크플로우"입니다. 일반 프롬프트가 매번 문맥을 새로 설명해야 하는 것과 달리, 스킬은 목적·절차·주의사항이 이미 문서화되어 있어 `/clone-site`, `/vibesec-scan` 처럼 한 줄만 입력해도 동일한 품질의 결과를 재현합니다.

## 프롬프트와 스킬의 차이

| | 프롬프트 | 스킬 |
|---|---|---|
| 재사용성 | 매번 새로 작성 | 이름으로 즉시 호출 |
| 일관성 | 작성자·순간마다 다름 | 문서화된 절차로 고정 |
| 개선 방식 | 대화 중 즉흥 수정 | 스킬 파일을 고쳐 팀 전체에 반영 |
| 축적 | 대화가 끝나면 사라짐 | 저장소에 남아 검색·재사용 가능 |

{{< callout type="info" >}}
"사이트 디자인 한 번에 클론 코딩"이나 "VibeSec-Skill로 OWASP 취약점 해소" 같은 한 줄 명령은 모두 이 표의 오른쪽 열, 즉 **스킬**에 해당합니다. 구체적인 예시는 [클론·생성 스킬](../../clone-and-scaffold/clone-site-design/)과 [보안·품질 스킬](../../security-and-quality/vibesec-owasp/)에서 다룹니다.
{{< /callout >}}

## 스킬이 재사용 가능한 이유

스킬은 보통 다음 세 가지를 하나의 파일(또는 폴더)에 함께 담습니다.

1. **트리거 조건** — 언제 이 스킬이 호출되어야 하는가 (예: "사이트 클론 요청이 들어오면")
2. **절차(runbook)** — 어떤 순서로 무엇을 확인하고 실행하는가
3. **주의사항·함정** — 과거에 실패했던 패턴과 그 회피 방법

이 세 가지가 문서로 고정되어 있기 때문에, 스킬을 실행하는 주체(사람이든 에이전트든)가 바뀌어도 결과 품질이 크게 흔들리지 않습니다.
