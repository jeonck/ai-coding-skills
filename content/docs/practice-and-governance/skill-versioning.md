한 줄 명령형 스킬이 진짜 자산이 되려면, 대화가 끝나도 사라지지 않고 **저장소에 코드처럼 남아 있어야** 합니다. 이 페이지는 스킬을 지속적으로 축적하고 개선하는 방법을 다룹니다.

## 스킬을 코드처럼 관리하기

1. **버전관리 시스템에 저장** — 스킬 정의 파일을 git 저장소에 커밋하고, 변경 이력을 남김
2. **알려진 함정을 스킬 안에 기록** — 실행 중 실패했던 사례와 원인을 스킬 문서에 직접 추가 (다음 실행에서 같은 실수를 반복하지 않도록)
3. **작게 시작해 반복 개선** — 처음부터 완벽한 스킬을 만들려 하지 말고, 실제로 여러 번 실행해보며 빠진 단계를 채워 넣음
4. **중복 스킬 통합** — 비슷한 목적의 스킬이 여러 개 생기면 하나로 합치고 나머지는 정리

{{< callout type="info" >}}
좋은 스킬 문서는 "무엇을 하는지"뿐 아니라 "과거에 어떤 함정이 있었는지"를 함께 담습니다. 예를 들어 이 사이트 자체를 만드는 스킬도 "pretty URL에서는 상대 링크 깊이가 소스 파일보다 한 단계 깊다"처럼 실제로 겪은 문제를 기록해 재발을 막습니다.
{{< /callout >}}

## 검색 가능하게 만들기

스킬이 수십 개로 늘어나면 이름만으로 찾기 어려워집니다. 아래처럼 최소한의 메타데이터를 함께 관리하세요.

- **트리거 키워드**: 어떤 말이 나오면 이 스킬을 떠올려야 하는가
- **한 줄 요약**: 이 스킬이 정확히 무엇을 하는가
- **최근 사용/개정 시점**: 오래 방치된 스킬은 최신 도구·정책과 어긋났을 가능성이 있음
