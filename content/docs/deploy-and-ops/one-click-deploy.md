"만들고, 저장소 만들고, 배포까지 완료"를 한 줄 명령으로 끝내는 스킬은, 실제로는 아래 단계를 순서대로 자동 실행합니다.

## 표준 절차

1. **원격 저장소 생성** — 조직/개인 계정에 적절한 이름으로 저장소 생성 (공개/비공개 확인)
2. **초기 커밋·푸시** — 생성된 코드를 커밋하고 원격에 푸시
3. **배포 설정 활성화** — GitHub Pages라면 Pages 소스를 GitHub Actions로 지정, Vercel이라면 프로젝트 연결
4. **CI 워크플로우로 빌드·배포** — [CI/CD 파이프라인 자동 구성](../cicd-skill/)에서 다루는 워크플로우 파일이 실제 빌드를 수행
5. **배포 URL 확인** — 실제로 페이지가 뜨는지 확인하고 사용자에게 URL 전달

{{< callout type="warning" >}}
저장소 생성이나 푸시처럼 외부에 노출되는 작업은 실행 전 사용자에게 저장소 이름·공개 범위를 확인받아야 합니다. "한 줄 명령"이라고 해서 확인 없이 공개 저장소를 만들어버리면 의도치 않은 정보 노출로 이어질 수 있습니다.
{{< /callout >}}

## 자주 발생하는 실패 지점

- **Pages 소스 미설정**: 워크플로우는 성공했는데 저장소 설정에서 Pages Source가 "Deploy from a branch"로 남아 있어 실제 사이트가 갱신되지 않는 경우
- **baseURL 불일치**: 정적 사이트 생성기의 `baseURL`이 실제 배포 경로(`/저장소명/`)와 다르면 CSS/링크가 깨짐
- **서브모듈 미포함**: 테마를 git submodule로 추가한 경우, CI 체크아웃 단계에서 `submodules: true`를 빠뜨리면 빈 테마로 빌드됨
