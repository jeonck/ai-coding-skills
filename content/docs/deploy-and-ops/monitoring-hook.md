배포가 "성공"으로 표시됐다고 해서 사용자가 실제로 정상적인 페이지를 보고 있다는 뜻은 아닙니다. 배포 직후 자동으로 상태를 확인하고, 이상이 있으면 즉시 알리는 훅이 필요합니다.

## 배포 후 자동 확인 항목

1. **헬스체크** — 배포된 URL에 실제로 요청을 보내 200 응답과 핵심 콘텐츠가 렌더링되는지 확인
2. **깨진 링크 스캔** — sitemap을 크롤링해 내부 링크가 모두 유효한지 확인
3. **핵심 지표 확인** — 응답 시간, 에러율처럼 이미 구축된 대시보드가 있다면 배포 시점과 연결해 급격한 변화 감지
4. **실패 시 알림** — 위 항목 중 하나라도 실패하면 팀 채널·이메일로 즉시 통보

{{< callout type="info" >}}
정적 사이트라면 `hugo server`로 로컬 빌드 후 sitemap의 모든 `<a href>`를 실제로 요청해보는 스크립트만으로도 배포 전에 깨진 링크를 대부분 잡아낼 수 있습니다. 이 확인은 배포 훅이 아니라 [CI/CD 파이프라인](../cicd-skill/)의 검증 단계에 넣는 것이 더 이릅니다.
{{< /callout >}}

## 알림은 조용하면 안 된다

{{< callout type="warning" >}}
모니터링 훅이 실패를 로그에만 남기고 아무도 보지 않는 채널로 보내면, 사실상 모니터링이 없는 것과 같습니다. 실패 알림은 팀이 실제로 확인하는 채널(예: 활성 Slack 채널)로 보내고, 반복 실패 시 에스컬레이션 규칙을 함께 정의하세요.
{{< /callout >}}
