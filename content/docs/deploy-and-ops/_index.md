만들고 검증한 결과물은 결국 실제 환경에 내보내야 의미가 있습니다. 이 섹션은 "한 줄 명령으로 배포까지 완료"하는 스킬과, 배포 이후 운영을 자동화하는 방법을 다룹니다.

{{< callout type="warning" >}}
배포 자동화는 되돌리기 어려운 작업(강제 푸시, 프로덕션 환경 변경)을 포함할 수 있습니다. 이 섹션의 스킬들은 속도만큼이나 "실패했을 때 무엇을 확인하고 어떻게 되돌리는지"를 함께 다룹니다.
{{< /callout >}}

## 이 섹션에서 다루는 내용

1. **한 줄 명령으로 GitHub Pages/Vercel 배포** — 저장소 생성부터 배포까지 한 흐름으로 연결하기
2. **CI/CD 파이프라인 자동 구성** — 빌드·테스트·배포 워크플로우를 스킬로 표준화하기
3. **배포 후 모니터링·알림 자동화** — 배포 성공/실패를 훅으로 감지하고 통보하기
