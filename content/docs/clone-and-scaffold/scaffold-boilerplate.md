새 프로젝트를 시작할 때마다 폴더 구조, 빌드 설정, CI 워크플로우, 배포 스크립트를 매번 손으로 맞추는 대신, "이 구조로 새 프로젝트 만들어줘" 한 줄로 표준화된 뼈대를 즉시 생성하는 스킬입니다.

## 스캐폴딩 스킬이 담아야 할 것

1. **디렉터리 구조 템플릿** — 실제 파일이 아니라 플레이스홀더가 포함된 `.template` 파일 형태로 보관
2. **필수 설정 파일** — 빌드 도구, 린터, 테스트 러너 설정을 조직 표준에 맞춰 미리 채움
3. **배포 파이프라인 골격** — GitHub Actions 등 CI/CD 워크플로우 파일까지 함께 생성
4. **채워야 할 값 목록** — 프로젝트명, 저장소 URL 등 스킬 실행 시 사용자에게 확인받아야 할 항목

{{< callout type="info" >}}
이 문서 자체도 그 예시입니다 — Hugo + Hextra 지식베이스 스캐폴딩 스킬은 `hugo.toml.template`, `home.md.template` 같은 템플릿 파일들과 "필요한 입력값 체크리스트"를 스킬 안에 미리 담아두고, 새 주제가 들어올 때마다 플레이스홀더만 채워 넣습니다.
{{< /callout >}}

## 좋은 스캐폴딩 스킬의 조건

- **플레이스홀더가 명확할 것**: `{{SITE_TITLE}}`처럼 눈에 띄는 형태로 표시해, 채우지 않은 값이 그대로 배포되는 사고를 방지합니다.
- **알려진 함정을 스킬 안에 문서화할 것**: 과거에 이 구조로 만들다가 겪었던 실수(내부 링크 깨짐, 인코딩 문제 등)를 스킬 문서에 남겨두면 다음 실행에서 반복하지 않습니다.
- **최소 구성으로 먼저 빌드 검증할 것**: 전체 콘텐츠를 다 채우기 전에 뼈대만으로 한 번 빌드해 구조적 오류를 먼저 잡습니다.
