디자이너가 Figma 같은 도구에 이미 만들어둔 화면을, 실제 프레임워크 코드로 변환하는 스킬입니다. "이 사이트처럼 만들어줘"([사이트 클론 코딩](../clone-site-design/))가 **살아있는 웹페이지**를 입력으로 받는다면, 이 스킬은 **디자인 파일**을 입력으로 받는다는 점이 다릅니다.

## 표준 절차

1. **디자인 파일에서 구조·토큰 추출** — 컴포넌트 계층, 색상/타이포그래피/스페이싱 변수
2. **기존 디자인 시스템과 매핑** — 이미 코드에 있는 컴포넌트가 있다면 새로 만들지 않고 연결(Code Connect)
3. **코드 생성** — 매핑되지 않은 컴포넌트만 새로 생성
4. **모션·인터랙션 반영** — 정적 레이아웃뿐 아니라 애니메이션 명세가 있다면 함께 구현
5. **화면 단위로 시각적 비교** — 디자인과 렌더링 결과를 나란히 확인

{{< callout type="info" >}}
이 흐름은 디자인 시스템 토큰과 컴포넌트 매핑이 미리 갖춰져 있을수록 정확도가 올라갑니다. 토큰 없이 하드코딩된 값으로 생성하면 이후 테마 변경이나 다크모드 대응 시 전부 다시 손봐야 합니다.
{{< /callout >}}

## 하드코딩을 피해야 하는 이유

디자인 목업의 색상 값을 코드에 그대로 박아 넣으면 당장은 동작하지만, 이후 브랜드 컬러가 바뀌거나 다크모드를 추가할 때 모든 파일을 다시 검색해 고쳐야 합니다. 디자인 시스템의 변수(토큰)를 그대로 코드의 변수로 연결하는 것이 이 스킬의 핵심 가치입니다.
