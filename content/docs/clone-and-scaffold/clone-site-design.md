"이 사이트처럼 만들어줘"라는 한 줄 명령이 실제로 동작하려면, 스킬 내부에 아래 절차가 고정되어 있어야 합니다. 이 페이지는 그 절차를 표준화한 예시를 다룹니다.

## 표준 절차

1. **대상 페이지 캡처** — 브라우저 자동화로 렌더링된 DOM, 스타일시트, 스크린샷을 함께 수집
2. **디자인 토큰 추출** — 색상·타이포그래피·간격을 재사용 가능한 변수로 정리
3. **컴포넌트 단위 분해** — 헤더/히어로/카드/푸터처럼 반복되는 블록 단위로 나누기
4. **대상 프레임워크로 코드 생성** — React/Next.js, Hugo, 정적 HTML 등 목표 스택에 맞게 생성
5. **로컬 빌드로 시각적 비교 검증** — 원본과 결과물을 나란히 스크린샷 비교

{{< callout type="info" >}}
Claude Code 환경에서는 `claude-in-chrome` 같은 브라우저 자동화 도구로 1단계를, `webapp-testing` 계열 스킬로 5단계를 수행할 수 있습니다. 실제로 손을 움직여 따라 해보고 싶다면 [Lab 1: 클론 코딩 스킬 만들기](../../../labs/lab1-clone-skill/)를 참고하세요.
{{< /callout >}}

## 저작권·라이선스 주의사항

- 디자인의 "그대로 복제"가 아니라 **구조와 패턴을 학습해 재구성**한다는 원칙을 지킵니다.
- 상업적으로 배포할 목적이라면 원본 사이트의 이용약관과 상표·저작권을 먼저 확인합니다.
- 클론 대상은 본인이 소유했거나 학습·데모 목적임이 명확한 사이트로 한정합니다.
