## 한 줄 명령

오른쪽 위 복사 아이콘을 눌러 그대로 붙여넣으세요. 별도의 스킬이나 훅이 미리 설정되어 있지 않아도, 이 문장 하나로 지금 열려 있는 PR을 바로 리뷰합니다.

```text
이 PR의 diff를 정확성·재사용성·효율성·테스트 커버리지 관점에서 리뷰하고, 발견한 문제를 "입력 X일 때 Y가 발생해 Z가 깨진다" 형태로 정리해서 코멘트로 남겨줘
```

매번 PR이 열릴 때마다 사람이 이 문장을 다시 입력하지 않으려면, [훅으로 반복 작업 자동화](../../docs/orchestration/hooks-automation/)를 참고해 PR 생성 이벤트에 이 지시문을 자동 연결하세요.

## 무엇이 일어나는가

1. PR 생성 이벤트를 훅이 감지
2. diff 범위 안에서 정확성·재사용성·효율성·테스트 커버리지 관점으로 점검
3. 발견된 문제를 "입력 X일 때 Y가 발생해 Z가 깨진다" 형태의 재현 가능한 시나리오로 정리
4. 가장 심각한 순서로 정렬해 PR 코멘트로 게시

## 사전 조건

- 리뷰 결과를 자동 승인/병합에 연결하지 않을 것 — 최종 판단은 항상 사람 리뷰어
- 훅 실행 시간이 길어지면 팀이 `--no-verify`로 우회하기 시작하므로, 실행 시간을 짧게 유지
