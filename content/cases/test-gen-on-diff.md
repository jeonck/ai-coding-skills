{{< callout type="info" >}}
**한 줄 명령**: "이 변경분에 대한 테스트 만들어줘, 수정 전 코드로 돌리면 실패하는 걸로"
{{< /callout >}}

## 무엇이 일어나는가

1. diff에서 새로 추가/수정된 함수와 분기 파악
2. 빈 입력, 경계값, 동시 요청 등 엣지 케이스 도출
3. 프로젝트에 이미 있는 테스트 프레임워크·네이밍 규칙을 그대로 준수
4. 생성한 테스트를 수정 전 코드에 돌려 실제로 실패하는지 먼저 확인
5. 수정 후 코드에 돌려 통과 + 커버리지 확인

## 사전 조건

- 데이터베이스·외부 API를 전부 mock 처리하지 말 것 — 이미 통합 테스트 관행이 있다면 그 방식을 우선 따를 것
- 아무것도 검증하지 않는 "통과만 하는" 테스트가 생성되지 않았는지 확인
